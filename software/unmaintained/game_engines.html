Project description

A game engine is a very helpful thing for game programmers. A simple game engine is just a library with game programming-related classes or functions.

I like emulation and old-school video games. I also like emulation-oriented portable game consoles. Some of these consoles use embedded Linux as firmware. I made my own multimedia library for some of these consoles. I want to test myself as a Linux programmer. This library has a cross-platform layer architecture. It has two kinds of subsystems: the system-specific and the common subsystem.
License

Features

A good game engine should have an optimal set of features. Let's look at the features of my game engines.

Common features:

  • Easy-to-use object-oriented API.
  • Graphics primitives.
  • Animated and static sprites.
  • Scrollable and static backgrounds.
  • Tile-maps.
  • Image transformation subsystem: resizing and mirroring.
  • Image loader with PCX and TGA support.
  • Text-drawing subsystem.
  • Custom fonts.
  • Timers.
  • Collision detector.
  • Integrated FPS counter.

Specific features of embedded Linux:

  • Simple input subsystem.
  • The display back-light control subsystem.
  • Fast render via frame buffer.
  • Low-level sound subsystem.
  • Simple sound mixer subsystem.
  • Audio loader with Waveform Audio support.
  • Simple audio-playing subsystem.
